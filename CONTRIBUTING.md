# Contributing to ClipVault

First off, thank you for considering contributing to ClipVault! It's people like you that make ClipVault a great tool.

## Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. We expect all contributors to maintain a respectful and welcoming environment.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for ClipVault.
- Ensure the bug was not already reported by searching on GitHub under Issues.
- If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion for ClipVault, including completely new features and minor improvements to existing functionality.
- Open a new issue and describe the feature or enhancement. Include as much detail as possible.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`xcodebuild test -scheme ClipVault -destination 'platform=macOS'`).
4. Ensure your code lints (`swiftlint`).
5. Issue that pull request!

## Development Setup
Please see the [README.md](README.md) for instructions on setting up the project locally.

## Security
If you discover a security vulnerability, please do NOT open a public issue. Instead, email security@clipvault.example.com.
